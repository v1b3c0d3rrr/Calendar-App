# Анализ социальных метрик по кластерам

**Дата**: 2026-03-11
**Выборка**: 121 токен-победитель (≥2x за 14 дней) по 5 кластерам типов пампа
**Источник данных**: CoinGecko API

---

## Основные выводы

### 1. Watchlist / Portfolio Users — НЕ дискриминирует

Медианное количество пользователей в watchlist варьируется между кластерами всего в **2.6x** (от 1,501 до 3,855). Это слабое различие, и — что важнее — **корреляция с мультипликатором отсутствует**: все квартили показывают примерно одинаковый медианный мультипликатор (~2.1x).

**Вывод**: количество пользователей в watchlist CoinGecko не является предиктором ни кластера, ни силы пампа.

### 2. Sentiment Votes — НЕ дискриминирует

Медианный sentiment = **100% bullish** для всех кластеров без исключения. Метрика полностью зашумлена — пользователи CoinGecko голосуют bullish за токены, которые уже растут.

Интересное наблюдение: токены с *низким* sentiment показывают *слегка более высокие* мультипликаторы (контрарианский сигнал?), но выборка слишком мала для статистически значимых выводов.

### 3. GitHub активность — варьируется, но не дискриминирует

Доля токенов с GitHub-активностью сильно различается по кластерам (от 2% до 42%), однако из предыдущего анализа известно, что GitHub **не отличает победителей от проигравших**. Это характеристика типа проекта, а не сигнал для входа.

### 4. Социальное присутствие — один выделяющийся кластер

**B_gradual_accumulation** выделяется: **94% Telegram** и **100% Twitter** — это токены с наиболее развитой социальной инфраструктурой. Для сравнения, A_deep_recovery имеет только 54% Telegram.

Это логично: постепенное накопление требует активного комьюнити, которое поддерживает интерес к проекту в период консолидации.

## Ограничения анализа

1. **Twitter followers** — CoinGecko не предоставляет количество подписчиков для большинства токенов. Доступна только ссылка на аккаунт (наличие/отсутствие).

2. **Отсутствие исторических данных** — все метрики снимаются на текущий момент, а не на момент начала пампа. Для ретроспективного анализа нужен LunarCrush (платный) или Twitter API.

3. **Реальные лидирующие социальные сигналы недоступны** через бесплатные API:
   - Скорость роста подписчиков (follower growth rate)
   - Упоминания KOL (Key Opinion Leaders)
   - Всплески социального объёма (social volume spikes)
   - Изменение тональности (sentiment shift)

4. **Контрарианский сигнал sentiment** — предварительное наблюдение (низкий sentiment → выше мультипликатор) требует проверки на большей выборке.

---

## Итог

Социальные метрики CoinGecko в текущем виде **не являются полезными предикторами** для скоринговой модели. Watchlist users и sentiment не дискриминируют ни между кластерами, ни между силой пампа. Единственное заметное различие — присутствие в Telegram для кластера B_gradual_accumulation, что может быть полезно как один из фильтров, но не как самостоятельный сигнал.

Для включения социальных сигналов в модель необходимы **платные источники данных** (LunarCrush, Twitter API, Santiment) с историческими рядами.
