# Команда /research <TICKER>

Полный цикл исследования с дискуссией. Цель — принять решение о входе ЗАРАНЕЕ, на leading indicators.

## Этап 1: Независимый ресёрч (3 параллельных анализа)
Собери данные о токене, затем последовательно проведи анализ от лица каждой персоны:

**CC (Cluster Analyst)** — по `personas/cc_persona.md`:
- Классификация кластера (cap tier + lifecycle)
- Ранние фильтры (Screens A-G)
- Кластерно-взвешенный скоринг (0-100)
- Нарративная фаза и тайминг
- Peer comparison + FDV benchmarking
- Двухбакетная оценка (A: fundamental, B: narrative/momentum)

**CG (Evidence Analyst)** — по `personas/cg_persona.md`:
- Evidence ledger с кросс-проверкой источников
- Конфликт-детекция (tolerance: price 5%, volume 15%)
- Gate checks по кластеру
- Детерминированный TA (HardSupport → PEZ → Stop → TP)
- 7-мерный скоринг (0-10)
- Verdict с conditions to upgrade/downgrade

**G (Due Diligence Analyst)** — по `personas/g_persona.md`:
- Ончейн-концентрация (Arkham clusters >5%)
- Верификация команды (LinkedIn, Tier-1 опыт)
- Fundraising history и инвесторы
- Red flags (unlock pressure, insider activity, low float)
- Zero-fluency анализ: только факты с источниками

## Этап 2: Структурированная дискуссия (минимум 10 раундов)

Формат каждого раунда:
```
## Раунд N: {тема}

**CC**: [позиция + аргумент + источник]
**CG**: [позиция + аргумент + источник]
**G**: [позиция + аргумент + источник]

Расхождения: [список]
Требуется уточнение: [что искать]
```

**Раунды 1-3: Презентация и выявление расхождений**
- Раунд 1: Каждый представляет свой вердикт и ключевые аргументы
- Раунд 2: Выявление конкретных точек несогласия (уровни, скор, риски)
- Раунд 3: Каждый задаёт уточняющие вопросы другим

**Раунды 4-6: Аргументация и поиск информации**
- Раунд 4: Ответы на вопросы с цитированием источников
- Раунд 5: Контраргументы по ключевым расхождениям
- Раунд 6: Поиск дополнительной информации для разрешения споров (WebSearch/WebFetch)

**Раунды 7-8: Сужение позиций**
- Раунд 7: Каждый пересматривает позицию с учётом аргументов других
- Раунд 8: Согласование уровней (entry, stop, TP1-3)

**Раунды 9-10: Финальная калибровка**
- Раунд 9: Финальные корректировки, проверка R:R
- Раунд 10: Формальный консенсус или фиксация разногласий

**Раунды 11+**: Если консенсус не достигнут — дополнительные раунды до разрешения.

ВАЖНО: В каждом раунде качество анализа должно улучшаться. Исследователи:
- Находят новые данные для подкрепления позиций
- Уточняют уровни на основе аргументов оппонентов
- Интегрируют перспективы друг друга
- Конкретизируют размытые тезисы

**⚠️ Anti-lookahead модерация:** Если исследователь ставит WATCH/AVOID на основании "нужно дождаться подтверждения объёмом / запуска продукта / стабилизации" — модератор ОБЯЗАН challenge: "Это lagging indicator. Оцени leading indicators и готовность войти с меньшей уверенностью + правильный стоп."
